Webpack: Carregando Imagens

Como processar e otimizar imagens com Webpack

Asset Modules

A partir do Webpack 5, não é mais necessário usar loaders externos como file-loader ou url-loader. O Webpack possui Asset Modules nativos para processar imagens e outros arquivos.

Tipos de Asset Modules

asset/resource

Emite um arquivo separado e exporta a URL. Substitui o file-loader.

asset/inline

Exporta data URI do asset. Substitui o url-loader.

asset/source

Exporta o código-fonte do asset. Substitui o raw-loader.

asset

Escolhe automaticamente entre exportar data URI ou arquivo separado.

Configuração no webpack.config.js

module.exports = {
    module: {
        rules: [
            {
                test: /\.(png|svg|jpg|jpeg|gif)$/i,
                type: 'asset/resource'
            },
            {
                test: /\.(woff|woff2|eot|ttf|otf)$/i,
                type: 'asset/resource'
            }
        ]
    },
    output: {
        assetModuleFilename: 'images/[hash][ext][query]'
    }
};

Como Usar

No JavaScript:

import Logo from './assets/logo.png';

const img = document.createElement('img');
img.src = Logo;
document.body.appendChild(img);

No CSS:

.logo {
    background-image: url('./assets/logo.png');
}

Otimização de Imagens

# Instalar image-minimizer-webpack-plugin
npm install image-minimizer-webpack-plugin imagemin --save-dev

# Para PNG
npm install imagemin-pngquant --save-dev

# Para JPEG
npm install imagemin-mozjpeg --save-dev

A otimização de imagens reduz o tamanho dos arquivos sem perder qualidade visual significativa, melhorando o desempenho da aplicação.
