Webpack: Múltiplas Entradas

Configurando múltiplos pontos de entrada e saída

Por que Múltiplas Entradas?

Em aplicações maiores, é comum ter diferentes páginas ou módulos que precisam de bundles separados. Múltiplas entradas permitem criar bundles específicos para cada parte da aplicação.

Entrada Única (Padrão)

module.exports = {
    entry: './src/index.js',
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
};

Múltiplas Entradas com Objeto

module.exports = {
    entry: {
        main: './src/index.js',
        admin: './src/admin.js',
        vendor: './src/vendor.js'
    },
    output: {
        filename: '[name].bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
};

// Resultado:
// dist/main.bundle.js
// dist/admin.bundle.js
// dist/vendor.bundle.js

Múltiplos Arquivos em Uma Entrada

module.exports = {
    entry: {
        app: ['./src/polyfills.js', './src/index.js'],
        vendor: ['react', 'react-dom', 'lodash']
    },
    output: {
        filename: '[name].bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
};

Com HtmlWebpackPlugin

const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    entry: {
        main: './src/index.js',
        admin: './src/admin.js'
    },
    output: {
        filename: '[name].[contenthash].js',
        path: path.resolve(__dirname, 'dist')
    },
    plugins: [
        // Página principal
        new HtmlWebpackPlugin({
            template: './src/index.html',
            filename: 'index.html',
            chunks: ['main']
        }),
        // Página admin
        new HtmlWebpackPlugin({
            template: './src/admin.html',
            filename: 'admin.html',
            chunks: ['admin']
        })
    ]
};

Otimização com SplitChunks

module.exports = {
    entry: {
        main: './src/index.js',
        admin: './src/admin.js'
    },
    output: {
        filename: '[name].[contenthash].js',
        path: path.resolve(__dirname, 'dist')
    },
    optimization: {
        splitChunks: {
            chunks: 'all',
            cacheGroups: {
                vendor: {
                    test: /[\\/]node_modules[\\/]/,
                    name: 'vendors',
                    priority: 10
                },
                common: {
                    minChunks: 2,
                    priority: 5,
                    reuseExistingChunk: true
                }
            }
        }
    }
};

Casos de Uso

🎯 Múltiplas Páginas (MPA)

Aplicações com diferentes páginas HTML, cada uma com seu próprio bundle JavaScript.

🔧 Separação de Vendor

Bibliotecas de terceiros em bundle separado para melhor cache no navegador.

⚙️ Admin vs Público

Separar código administrativo do código público da aplicação.

📱 Diferentes Dispositivos

Bundles específicos para mobile, desktop ou diferentes plataformas.

Vantagens

  • Melhor cache no navegador
  • Carregamento mais rápido de páginas específicas
  • Código compartilhado automaticamente
  • Facilita manutenção de grandes aplicações
  • Lazy loading mais eficiente
