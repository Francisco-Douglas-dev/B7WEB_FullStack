Webpack: Refatorando Output

Boas práticas para organização avançada do output

Por que Refatorar o Output?

À medida que a aplicação cresce, é essencial ter uma estrutura de output bem organizada. Isso facilita o deploy, melhora o cache do navegador e torna a manutenção mais simples.

Antes da Refatoração

// Estrutura desorganizada
dist/
├── main.js
├── vendor.js
├── logo.png
├── style.css
└── font.woff2

Depois da Refatoração

dist/
├── js/
│   ├── main.[contenthash].js
│   ├── vendor.[contenthash].js
│   └── runtime.[contenthash].js
├── css/
│   ├── main.[contenthash].css
│   └── vendors~main.[contenthash].css
├── assets/
│   ├── images/
│   │   ├── logo.[hash].png
│   │   └── icons/
│   └── fonts/
│       └── roboto.[hash].woff2
└── index.html

Configuração Completa Refatorada

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const { CleanWebpackPlugin } = require('clean-webpack-plugin');

module.exports = {
    mode: 'production',
    entry: {
        main: './src/index.js',
        vendor: './src/vendor.js'
    },
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: 'js/[name].[contenthash:8].js',
        chunkFilename: 'js/[name].[contenthash:8].chunk.js',
        assetModuleFilename: 'assets/[name].[hash:8][ext]',
        clean: true
    },
    module: {
        rules: [
            // CSS/SCSS
            {
                test: /\.(sa|sc|c)ss$/,
                use: [
                    MiniCssExtractPlugin.loader,
                    'css-loader',
                    'sass-loader'
                ]
            },
            // Imagens
            {
                test: /\.(png|jpg|jpeg|gif|svg)$/i,
                type: 'asset/resource',
                generator: {
                    filename: 'assets/images/[name].[hash:8][ext]'
                }
            },
            // Fontes
            {
                test: /\.(woff|woff2|eot|ttf|otf)$/i,
                type: 'asset/resource',
                generator: {
                    filename: 'assets/fonts/[name].[hash:8][ext]'
                }
            }
        ]
    },
    plugins: [
        new CleanWebpackPlugin(),
        new HtmlWebpackPlugin({
            template: './src/index.html',
            filename: 'index.html',
            minify: {
                removeComments: true,
                collapseWhitespace: true
            }
        }),
        new MiniCssExtractPlugin({
            filename: 'css/[name].[contenthash:8].css',
            chunkFilename: 'css/[name].[contenthash:8].chunk.css'
        })
    ],
    optimization: {
        moduleIds: 'deterministic',
        runtimeChunk: 'single',
        splitChunks: {
            cacheGroups: {
                vendor: {
                    test: /[\\/]node_modules[\\/]/,
                    name: 'vendors',
                    chunks: 'all',
                    priority: 10
                },
                common: {
                    minChunks: 2,
                    priority: 5,
                    reuseExistingChunk: true,
                    enforce: true
                }
            }
        }
    }
};

Boas Práticas

🔹 Use ContentHash para Cache

ContentHash garante que apenas arquivos modificados tenham novos nomes, otimizando o cache do navegador.

🔹 Separe por Tipo de Asset

Organize arquivos em subpastas (js/, css/, assets/) para facilitar localização e manutenção.

🔹 Runtime Chunk Separado

Extrai o runtime do Webpack em arquivo separado para melhor cache de vendors.

🔹 Clean Plugin

Limpa a pasta dist automaticamente antes de cada build, evitando arquivos obsoletos.

🔹 Hash com Tamanho Fixo

Use [contenthash:8] para hashes curtos mas únicos, reduzindo o tamanho dos nomes de arquivo.

Resultados da Refatoração

  • Cache otimizado - apenas arquivos alterados são baixados novamente
  • Deploy mais eficiente - estrutura previsível e organizada
  • Debugging facilitado - fácil localizar arquivos específicos
  • Performance melhorada - chunks otimizados e lazy loading
  • Manutenção simplificada - estrutura clara e lógica
