Webpack Config

Arquivo de configuração webpack.config.js

webpack.config.js

O arquivo webpack.config.js é o coração da configuração do Webpack. Ele permite personalizar completamente o processo de build da sua aplicação.

Estrutura Básica

const path = require('path');

module.exports = {
    // Modo de compilação
    mode: 'development', // ou 'production'
    
    // Arquivo de entrada
    entry: './src/index.js',
    
    // Configuração de saída
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: 'bundle.js'
    }
};

Principais Propriedades

mode

Define o ambiente: development ou production

entry

Ponto de entrada da aplicação. Pode ser uma string, array ou objeto.

output

Define onde e como os arquivos gerados serão salvos.

module

Configura loaders para processar diferentes tipos de arquivos.

plugins

Extensões que realizam tarefas adicionais no processo de build.

Exemplo Completo

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    mode: 'development',
    entry: {
        main: './src/index.js',
        vendor: './src/vendor.js'
    },
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: '[name].bundle.js'
    },
    module: {
        rules: [
            {
                test: /\.css$/,
                use: ['style-loader', 'css-loader']
            }
        ]
    },
    plugins: [
        new HtmlWebpackPlugin({
            template: './src/index.html'
        })
    ]
};
