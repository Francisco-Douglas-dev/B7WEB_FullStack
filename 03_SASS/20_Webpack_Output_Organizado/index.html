Webpack: Output Organizado

Estruturando a saída dos arquivos compilados

Organização do Output

Organizar adequadamente os arquivos de saída do Webpack facilita a manutenção, deploy e otimização da aplicação. Uma estrutura bem definida melhora o cache e performance.

Configuração Básica de Output

const path = require('path');

module.exports = {
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: 'bundle.js',
        clean: true  // Limpa a pasta dist antes de cada build
    }
};

Output Organizado por Tipo

module.exports = {
    output: {
        path: path.resolve(__dirname, 'dist'),
        // JavaScript em subpasta com hash
        filename: 'js/[name].[contenthash].js',
        // Assets em subpasta específica
        assetModuleFilename: 'assets/[name].[hash][ext]',
        clean: true
    },
    
    module: {
        rules: [
            // Imagens na pasta images/
            {
                test: /\.(png|jpg|jpeg|gif|svg)$/i,
                type: 'asset/resource',
                generator: {
                    filename: 'images/[name].[hash][ext]'
                }
            },
            // Fontes na pasta fonts/
            {
                test: /\.(woff|woff2|eot|ttf|otf)$/i,
                type: 'asset/resource',
                generator: {
                    filename: 'fonts/[name].[hash][ext]'
                }
            }
        ]
    }
};

Padrões de Nomenclatura

[name]

Nome do arquivo original ou nome do chunk

[contenthash]

Hash baseado no conteúdo do arquivo (melhor para cache)

[chunkhash]

Hash baseado no chunk específico

[hash]

Hash da compilação completa

[ext]

Extensão do arquivo original

Plugin para CSS Separado

const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
    plugins: [
        new MiniCssExtractPlugin({
            filename: 'css/[name].[contenthash].css'
        })
    ],
    module: {
        rules: [
            {
                test: /\.css$/,
                use: [
                    MiniCssExtractPlugin.loader,
                    'css-loader'
                ]
            }
        ]
    }
};

Benefícios

  • Cache otimizado no navegador
  • Estrutura organizada e previsível
  • Facilita versionamento e rollback
  • Melhor separação de responsabilidades
  • Deploy mais eficiente
